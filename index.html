Android Thingsはおもしろそうなので調べてみようという話

May 24th, 2018 1:15 pm

Wireless Japan 2018に行ってきまして、色々とお勉強になりそうなネタを得たのでめもめも。

先日のGoogle I/OでAndroid Things 1.0が宣言されたとのことでした。自分のアンテナの低さに驚きつつ、ワクワクしながらAndroid技術者の方々のセミナーを聞いてきました。

Android Things x Kotlin x Deep Learning

Android ThingsとはGoogleが提供するIoT向けプラットフォーム。

  • Android Things

    • AndroidのAPIが使える
    • 4つの生産PF／2つの開発PF　ラズパイ含む
    • カーネル、ファームウェアはGoogleが開発
    • 周辺機器IF　Peripheral I/O API　GPIO/PWM/I2C/SPI/UART
    • Google Play Services/Firebase/Google Cloud
    • 一貫性、コスト、セキュリティ->解決！

  • Kotlin

    • Androidの標準プログラミング言語
    • Javaライブラリ互換
    • 静的型付け言語
    • オブジェクト指向
    • 関数型プログラミング
    • 安全(?)

  • Android Things x Kotlin x Deep learning

    • TensorFlow Lite
    • Edge Computing cf. Cloud Computing

  • （例）テレビ漬け防止強制OFFガジェット

    • Android Studioでプロジェクト作成
    • 転移学習（Transfer Learning）
    • TensorFlow Liteにモデルを転送

NFCタグが最近かなり身近になったという話

1枚50~60円のNFCタグを色んなところに貼りまくって、AndroidスマホのNFCリーダで、予め書き込んだURLを読み取って色々できる。

子どもの遊びとの親和性がめちゃ高いようです。おもちゃにタグ貼り付けて2歳の子どもがスマホで読み取ってリンクをタップしてYoutube動画を観る、なんていうシーンは感動すら覚える。

中国のQRコード社会現象が盛り上がってる中、価格が更に下がれば、まだまだ可能性があるなあと思いました。

IoTで仮説検証をさくっとやりましょうという話

沖縄に移住したスピーカーの方が、高温多湿の生活で不健康にならないよう、既存のサービスでかなり簡単に温湿度モニタリングができちゃうようにしちゃったという話からスタート。

嫌いではないけれど精神的ハードルがあるハンダ付け。ハンダ付けなしでやってみようという制約を自らに課し、Google Spreadsheet、RasPi Zero W（WiFi、BLE対応。これ欲しい！）、WxBeacon2で作っちゃった。

これだけだとAndroid感がないが、次の話では、Pimoroni Rainbow HAT for Android Things + Raspberry PiというAndroid Things対応の拡張ボードと、Wio Nodeというボードを使った、キティちゃんのM&Msディスペンサーを作ったという話を聞けた。ハッカー感がスゴイ。

IoT（=ものづくりの手段）自体をもっと手軽にしたいという話

有名な方らしいが、超熱い、「IoTをもっと手軽にしたい！今後は全ての企業がIT/IoT企業になる！」と語っていらした。

自社のプロダクトである、NefryとかWaffleとかいうデバイスが販売中らしいのですが、プレゼンの中で仕様とオモロイ点を聞き逃してしまったので、このあと改めてブースに立ち寄るか、調べてみたいと思う。

感想めいたもの

2年ほど前、Arduinoと土壌センサーで観葉植物IoTをやってみたことがあるが、こういったことがさくっとできてしまう色んなサービスやデバイスが増えてきているという印象を強く受けました。

これからの子どもたちがとっつきやすい教育寄りのプロダクトもかなり出てきている。息子たちにもいつか遊ばせてやりたい、と率直に思いました。まず何か自分がさくっと作って、彼らに遊んでもらって、自然と彼らも作りたくなって、みたいな流れが理想です。

とりあえず、

  • Android Things
  • TensorFlow Lite
  • Raspberry Pi Zero W
  • RxBeacon2
  • Wio Node

あたりはさくっと試してみたいと思います。そしてまたここに結果を書き残してみたい思います。

My仮想通貨を60分で作ってみた話

May 23rd, 2018 7:53 pm

ネットニュースの仮想通貨やICOの話が、イマイチ頭に染み込まない、自分の言葉で説明できない今日この頃。ハンズオンで自分の仮想通貨ができてしまうというステキな勉強会を開いてくれるというので、雨の渋谷に行ってきました。

結論、めちゃおもろい。

ツールも既に結構揃ってて（ほとんどアプリもドキュメント英語だけど）、コマンドライン打てなかったり、さくさくコード書けなくてもある程度まではいじいじできることがわかりました。

頭に残ったキーワード

ブロックチェーン（特にEthereum）を説明するに不可欠なキーワードなどなど。

  • Wallet　秘密鍵の管理ツール
    • MetaMask
    • CiperBrowser
    • MyEtherWallet ＊フィッシングサイト多数なので注意。念の為日本人ブロガーさんを経由。
  • Network　ブロックチェーンネットワーク
    • Ropsten
  • Address　ブロックチェーン上の自分の位置情報のようなもの
  • Vault　
  • Faucet
  • ETH/Wei　ethereumの通貨単位
  • remix.ethereum.org　
  • etherscan　自分のアドレスに紐づく多くの情報が可視化されるサイト
  • PoW　コンセンサスアルゴリズムの一種
  • Transaction(Tx)　
  • Contract
  • Token
  • Dapp　ブロックチェーン上で開発できるアプリケーション
  • Solidity　Dappを記述するJS由来の言語

まだ自分の言葉で表現できないものが多いので、引続き学習します。

印象に残った話

  • スタートアップ界隈ではブロックチェーンを使ったプロダクトやサービスがコンテストに参加したりしているが、「それって本当にブロックチェーンじゃなきゃだめなの？」って話に必ずなる
  • スマートコントラクトについて、自動販売機のアナロジーはよく聞かれるが、詳しい人から見ても、わかりやすい良い例とのこと（誰が言い始めたんだろう）
  • マイナーさんはMainnet/Testnetに関わらず、トランザクションの手数料でも稼いでいる
  • ICOで上手く行っている例はまだ聞いたことがない（このあたりは事例を掘ってみると良い）

深めるといいことあるかもなポイント

  • Dapp開発。これで具体的に何ができるようになるのか、WWWにおけるWebAppとの差異、強みが知りたい。
  • 今のBlockchainは、WWWの黎明期の状態に似ている。DARPANETとかインターネットの夜明けに関するストーリーを知りたい。

まとめと感想

新しい高度なおもちゃを得た気分です。自分自身、あまり仮想通貨には興味はなくて（ただし、新しい通貨が生まれ、人の間で価値を持ち始めるプロセスはめちゃめちゃオモロイので、それはウォッチし続けたいし、経済学などで大昔から研究されているはずなので、別途勉強したい）、今までにない、新しい発想のテクノロジーであるところに惹かれました。

これは当分がっつり遊べるな、と。

ただ、それを世の中に役立ったり、マネタイズできる糸口まだ見つかっていない気もします。Decentralizedであることが価値になるようなビジネスとは何か。それを理解しない限り、「それブロックチェーンである必要あるの？」ということになり、自然と人々が離れていってしまう。

まだ薄っぺらーい知識しかないので、これからザクザク掘り下げていきたいと思います。とにかくDappはひとつふたつ作ってみたい！

という感じでした。

OctopressでGithub Pagesにブログをポストするまでの話

May 22nd, 2018 2:05 pm

普段自分にインプットしている情報はおそらく、

  • そのまま腹落ちさせてじわじわと活用する大器晩成型
  • すぐにアウトプットすることが、未来の自分を含めた誰かの価値になる即戦力型

に分かれる、と考えています。

そのうち、後者を実現するメソッドを探していました。SNSも多様化しているので、何か既存のプラットフォームでやるならどれがいいかなーと軽く考えたのですが、「ベタだけど、やっぱブログだな」、ということになりました。自分会議。

ブログも色々あるようですが、このOctopressが面白そうなので、ホリデープログラマーよろしく、たまーにいじくりながら、好きなことを書き書きしてみたいと思います。

とりあえず、まずはこのOctopressでブログ投稿に必要な環境構築にやったことをめもめも。

Octopressとは

Markdownでラクに書けるブログジェネレータ。Github Pagesにホスティングさせれば無料公開できるだけでなく、バージョン管理もワンストップにできる。

http://octopress.org/ https://pages.github.com/

必要な知識

  • CUI
  • git/Github
  • markdown記法

セットアップ

Github Pagesとの連携

あらかじめGithubでレポジトリttskonno.github.ioを作成し、レポジトリのURLをコピペ。

デプロイ

ちなみにデプロイはrake gen_deployというコマンド1つでも実行できます。

記事の投稿

これでひととおりブログを投稿するまではできます。他にも_config.ymlを編集すればAuthorやブログタイトルの編集なども可能です。

テーマは3rd party Octopress Themesを参照。コマンドで簡単に変更できます。もう少しバリエーションがほしいところ。あまりユーザーが居ないからなのかな。

セットアップは少しハマったけど、Github Pagesにpushできたらあとは簡単。簡便化がOctopressを選んだ目的ではないが、少し自動化できるところがあれば、スクリプトを組んでみようかなと思っています。
